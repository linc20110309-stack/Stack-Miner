# AI_DEV_PROMPT.md

# 奇妙寻宝（Stack Miner）AI开发总规范

Version: 2.0

---

# 1. 文件定位

本文件是整个项目的最高级规范。

所有 AI（Claude Code、Cline、GPT、Gemini、DeepSeek 等）在生成任何代码之前：

必须完整阅读：

```text
AI_DEV_PROMPT.md
GAME_DESIGN.md
UI_ART_STANDARDS.md
DEV_STANDARDS.md
```

并严格遵守。

优先级：

```text
AI_DEV_PROMPT.md
    ↓
GAME_DESIGN.md
    ↓
UI_ART_STANDARDS.md
    ↓
DEV_STANDARDS.md
```

若出现冲突：

高优先级覆盖低优先级。

奇妙寻宝（Stack Miner）


# 3. 产品原则

AI生成任何功能时必须遵守：

## 原则1：简单优先

优先：

```text
简单
直接
可理解
```

不要设计：

```text
复杂经济系统
复杂成长系统
复杂装备系统
复杂任务系统
```

---

## 原则2：轻量优先

目标包体：

```text
< 10MB
```

优先：

```text
Canvas绘制
Emoji
系统字体
```

避免：

```text
大量图片
大量动画资源
大型第三方库
```

---

## 原则3：小游戏优先

必须适配：

```text
微信小游戏
抖音小游戏
```

其次：

```text
H5
```

不得优先考虑：

```text
PC网页
桌面程序
```

---

## 原则4：可玩优先

任何设计：

优先保证：

```text
能玩
流畅
无死局
```

再考虑：

```text
特效
美术
扩展功能
```

# 6. AI开发原则

## 原则1

一次只开发一个模块。

禁止：

```text
同时重构十几个文件
```

---

## 原则2

先保证运行。

再优化结构。

---

## 原则3

发现规范冲突时：

优先参考：

GAME_DESIGN.md

---

## 原则4

不得擅自新增玩法。

例如：

用户没有要求：

```text
炸弹
火箭
技能
道具
```

AI不得自行添加。

DEV_STANDARDS.md

要求。


# 9. 输出规范

AI输出代码时：

必须：

## 输出完整文件

正确：

```text
utils/backpack.js
完整代码
```

错误：

```text
只贴修改部分
```

---

## 输出中文注释

所有核心函数：

必须包含：

```javascript
/**
 * 函数说明
 */
```

---

## 输出可运行代码

禁止：

```text
TODO
以后实现
自行补充
伪代码
```

---

# 10. Bug修复原则

发现问题：

优先：

```text
修复
```

不要：

```text
推翻重写
```

除非：

当前架构已经无法维护。

---

# 11. AI自检

生成代码后必须检查：

## 功能

* 是否符合 GAME_DESIGN

## UI

* 是否符合 UI_ART_STANDARDS

## 工程

* 是否符合 DEV_STANDARDS

## 兼容

* 是否兼容微信小游戏

## 性能

* 是否能稳定运行

---

# 12. 最终目标

AI的一切开发行为都应围绕：

```text
简单
轻量
稳定
好玩
可上线
```

五个目标展开。

若某个设计虽然高级，

但会增加复杂度，

则放弃高级设计，

优先选择：

最简单且稳定的方案。
